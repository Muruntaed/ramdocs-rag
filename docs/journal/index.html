Changelog

v1.0 → v2.0 (2026-05-15)

Changes: the analyzer now returns a mandatory entity field; cosine clustering is replaced by deterministic entity grouping with substring merging; the mediator is invoked only to resolve intra-group disagreement; the final answer is a list of variants (one per entity group above the 0.40 weight threshold).

Expected: Recall@gold ≥ 0.75 and misinfo-rejection holding flat.

Observed: EM-any-gold +0.500, Recall@gold +0.077, noise-rejection +0.264, cost −55% — large wins. Misinfo-rejection −0.125 and citation-faithfulness −0.122 — misinfo with a fabricated entity now forms its own group and passes the threshold. Addressed in v3.

v2.0 → v3 (2026-05-15)

Changes: a single-pass Skeptic agent is added after the mediator. It sees every draft variant plus the entire document pool and returns a keep / reject verdict per variant based on four checks — entity grounding, citation faithfulness, counter-evidence and a depth-asymmetry rule that flags thin-stub variants when other variants have richer corroboration. A pipeline-level fallback restores the draft if the Skeptic ever rejects every variant. The entity-group threshold is loosened from 0.40 to 0.30 to lift multi-answer recall.

Expected: Misinfo-rejection back to ≥ 0.85 with no major loss on EM or Recall.

Observed: Misinfo-rejection 0.792 → 0.875 (+0.083), Citation-faithfulness +0.024, F1 essentially flat (+0.005), Recall −0.021, EM −0.041, abstention 0. v3 is the recommended default when robustness matters; v2.0 stays the cheapest and best-EM option.

Internal iterations toward v3 (the aggressive v3.0 and the over-conservative v3.1) live in the repository under pipelines/v3_0_skeptic/ and pipelines/v3_1_conservative_skeptic/ for code archaeology but are intentionally omitted from this public journal. See runs/_journal.md for the full internal log.

v3 → v4 · Evidence Evaluator (2026-05-16) — recommended default

Changes: three layers stacked on top of v3. (1) A per-document Evidence Evaluator agent emits a structured DocTrust report — four local quality axes (internal_consistency, encyclopedic_quality, specificity, relevance), a composite trust_score and a closed-enum list of red_flags — and the trust signal feeds a new W_TRUST = 0.35 slot in the reliability formula, replacing the dormant recency / authority placeholders of v1–v3. (2) A shared safety module (core/safety.py) is auto-attached to every prompt across all versions (grounding + prompt-leak refusal + output-schema lock; the mediator additionally gets off-topic refusal), with an agent_probe CLI for ~$0.0005 single-agent debugging. (3) The analyzer prompt is rewritten with a top-anchored HARD GROUNDING RULE (anti-paraphrase / anti-recency / anti-entity-mismatch + a verbatim-quote test), GOOD/BAD contrasts on RULE A1 (citation-vs-description) and RULE A2 (pure-table fallback), a mandatory PLACE-DISAMBIGUATOR for towns / CDPs / villages, a HARD CHECKLIST expanded 5 → 7, and a role-question + category-verbatim-scope patch with flexible TEXT FORMAT (bare noun OR short noun-phrase) for in-scope category-noun answers.

Expected: recover v3 anchors and improve beyond. All five pre-declared anchors must hit on two consecutive freeze runs: Abstention 0.000, Misinfo-rejection ≥ 0.917, Noise-rejection ≥ 0.972, EM-substring ≥ 0.917, F1-multi ≥ 0.77.

Observed: EM-any-gold 0.875 (+0.083 vs v3), F1-multi 0.818 (+0.143), Misinfo-rejection 0.917 (+0.042), Recall@gold 0.847 (+0.201), Citation-faithfulness 0.820 (+0.069), Abstention 0.000. Run-to-run agreement is bit-identical on Abstention, Misinfo-rejection, Noise-rejection and Correct-citation; EM-any shifts by exactly +1 question in has_noise between freezes (within the ±1/12 stability rule). Cost / latency lift by ~2× vs v3 (~$0.022/Q, 24 s/Q vs ~$0.010/Q, 13.5 s/Q). v4 · Evidence Evaluator is the recommended default; v3 stays the cheap option when budget matters more than the last percentage points.

Internal iterations toward v4 (an initial draft of the analyzer prompt that under-enforced grounding and regressed quality, plus smoke v2 / v3 / v4 / v5 probe series and a post-AB-fix freeze) live in runs/v4.1_promptfix/ and runs/_journal.md for code archaeology; only the final two freeze runs feed the published baseline.
